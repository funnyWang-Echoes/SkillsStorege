Abstract

§Abstract · 段落 1

中文翻译： 由于难以规模化，生成对抗网络（GAN）在文本条件图像合成任务上似乎正在失宠。而稀疏激活的混合专家模型（sparsely-activated mixture-of-experts, MoE）近来被证明是一种在有限算力下训练大规模模型的有效方案。受此启发，我们提出 Aurora——一种基于 GAN 的文本到图像生成器，它用一组 expert 学习特征处理，配合一个稀疏 router 为每个特征点选最合适的 expert。为了忠实解码采样随机性和文本条件到最终合成结果，我们的 router 在决策时自适应地考虑了文本集成的全局潜码。在 64×64 分辨率下，我们的模型在 LAION2B-en 和 COYO-700M 上训练，在 MS COCO 上达到 6.2 的 zero-shot FID。我们发布了代码和 checkpoints 以推动社区进一步发展。

§1.2 · Introduction · 段落 2（GAN 的问题与 T2I 作为翻译任务）

中文翻译： 本工作把焦点放回 GAN，它常被批评训练不稳定、模式覆盖有限。但我们认为，T2I 生成更像一个翻译任务（即把一个有信息的数据分布迁移到另一个），而不是纯生成任务（即用随机噪声作为输入建模观测分布），因此引入文本条件本身就能在一定程度上缓解上述两个问题——这一点也被一些近期研究实证支持。即便如此，模型规模难以扩大仍是 GAN 的主要缺点。具体来说，不同于扩散模型通过迭代过程生成图像，GAN 的生成器典型地用一个前馈网络从采样的潜码渲染出图像。考虑到实际中有限的计算资源（如 GPU 显存），直接学一个足够大的前馈模型仍然有挑战。

§2.1 · Related Work · 段落 1（GANs）

§2.2 · Related Work · 段落 2（Text-conditioned image synthesis）

§2.3 · Related Work · 段落 3（Mixture-of-Experts）

中文翻译： 与 ensemble 共享类似哲学，MoE 学习一组 expert 模块处理输入信号，加上一个 gating 模块（也称 router）帮助组合不同 expert 的输出。在这种设计下，每个 expert 被期望聚焦于输入空间某个子区域的学习，而 router 负责跨 expert 的权重分配。稀疏激活 MoE 因其离散路由机制区别于其他 MoE 变体——通过从大候选池中为每个输入样本只选一个或几个 expert，sparse MoE 提供了以几乎不增加计算代价扩大模型规模的可行方案。本工作中，我们将 sparse MoE 的思想引入 GAN 生成器设计，并通过考虑文本集成的随机性，为 T2I 任务适配路由机制。

§3.1 · Text-conditioned Sampling · 段落 1（CLIP + mapping network）

中文翻译： 如 StyleGAN 所指出，对潜码 z 应用可学习的基于 MLP 的 mapping network，可以得到比原生潜空间 Z 更解耦的潜空间 W，已被证明有利于语义操控。在 T2I 设定下，我们选择也将文本条件送入 mapping network（跟随前作 [29, 59]），使采样过程更好地感知文本信息。具体来说，我们采用 CLIP 作为文本编码器，将输入文本 c 转换为 token 序列 t_seq，以及一个聚合整句信息的全局 token t_g。然后我们将全局 token 拼接到采样的潜码上做进一步处理：

$$ \{t_{seq}, t_g\} = \text{CLIP}(c) \quad (1) $$
$$ w = \text{MLP}(\text{concat}(z, t_g)) \quad (2) $$

其中 w 表示解耦的潜码。这里需要指出，考虑到视觉-语言对齐与 T2I 生成之间的任务差距，我们冻结 CLIP 的文本编码器，并在编码器顶部堆叠更多可学习层以允许灵活调优 [29]。

即时分析： - 这段在说什么：文本条件的两条路径——全局 token t_g 走 mapping network 和 z 一起变 w；token 序列 t_seq 留给后面 §3.2 的 cross-attention - 【回应 Q7】：冻结 CLIP 是因为任务 gap（对齐 vs 生成），堆可学习层是为了适配生成任务。具体几层在代码里：run_synthesize.py:113 的 num_layers_text_enc: 4。Q7 已回答 ✓ - 【呼应 §3.0】：§3.0 的 z ∈ R^512 在这里进入 mapping。注意代码里 z_dim=128，w_dim=1024，与论文数字不一致 - 有没有问题：mapping network 同时吃 z 和 t_g，这意味着 w 里同时编码了"采样随机性"和"文本条件"——这正是 §1.4 说的"文本集成的全局潜码"。设计自洽

中文翻译： 卷积块。潜码调制（latent modulation）从 adaptive instance normalization（AdaIN）发展而来，在生成领域已证明其有效性，因而是 GAN 模型的基础操作。该技术通过 demodulation 融入卷积操作（即 modulated convolution, ModConv）进一步改进，并通过配备可学习的特征形变（即 modulated transformation module, MTM）再进一步。我们在卷积块中用两个 MTM，并用 skip connection 构造该块。因此卷积块的输出为：

§3.2 · Unit Generative Block · 段落 3（Attention block 结构）

中文翻译： 注意力块。从公式 (3) 可知，卷积块中文本条件 c 只通过全局 token t_g 以公式 (2) 的 w 为中介与生成特征交互。为也充分利用 token 序列 t_seq，我们引入注意力块用于文本-图像交互，这在多模态学习中被广泛使用。具体来说，注意力块由一个 self-attention 层、一个 cross-attention 层和一个 feedforward network（FFN）组成。因此有：

$$ f_{proj} = \text{ModConv}(f_{conv}, w) \quad (4) $$
$$ f_{sa} = \text{SelfAttn}(f_{proj}) + f_{proj} \quad (5) $$
$$ f_{ca} = \text{CrossAttn}(f_{sa}, t_{seq}) + f_{sa} \quad (6) $$
$$ f_{ffn} = \text{FFN}(f_{ca}) + f_{ca} \quad (7) $$
$$ f_{out} = \text{ModConv}(f_{ffn}, w) \quad (8) $$

其中公式 (4) 和 (8) 中的 modulated convolution 用作特征投影器以调整通道数。公式 (7) 中的 FFN 用两层 MLP 独立处理每个特征点。这里我们使用 ℓ2-distance attention [29] 来稳定训练。

§3.2 · Unit Generative Block · 段落 4（Multiple experts and sparse routing —— 核心机制）

中文翻译： 多专家与稀疏路由。回想本工作的主要动机是扩大 GAN 生成器的容量。一个直接方案是让网络更深或更宽，但这可能导致显存溢出或训练不稳定。受 NLP 实践启发，我们将 sparse MoE 的哲学融入单元生成块设计以扩大生成器容量。具体来说，我们把公式 (7) 中的 FFN 视为一个 expert，它专门处理单点特征，然后总共 engage N 个 expert {FFN_j}_{j=1}^N，形成候选池。接着我们用稀疏 router 为每个特征点选最合适的 expert。不同于现有 sparse MoE 方法，我们的 router 不仅基于输入特征还基于文本集成的全局潜码 w 自适应决策，这更好地匹配 T2I 生成的目标（即解码文本条件和采样随机性到图像）。在此路由策略下，公式 (7) 可重写为：

举一反三

  • 机制迁移：Aurora 的 router 在选 expert 时同时考虑"局部特征点"和"全局条件 w"——这种"局部-全局联合路由"思路可以迁移到 3D 生成（如 NeRF/Gaussian Splatting 的 MoE），因为 3D 体素的语义类别（地面、建筑、天空）既依赖局部体素特征，也依赖全局场景条件（室内/室外、风格）。来源：§3.2 段落 4 公式 (9) + §4.4 段落 1 路由可视化验证
  • 方法改进：Aurora 发现 T2I GAN 的潜空间不光滑（§4.2 段落 3 + §4.4 段落 2），且假设原因是 PPL 缺失和 cross-attention 主导。一个直接的改进方向是：把 PPL 正则加回来，或者把 cross-attention 的 t_seq 注入改造成"平滑调制"而非"token 级 attention"，看潜空间能否恢复平滑。这是验证 §4.4 假设的最小实验。来源：§4.2 段落 3（反直觉发现）+ §4.4 段落 2（假设）
  • 生态启示：Aurora 的核心贡献不只是 MoE 本身，而是"开源了一个 1.16B 的 GAN T2I baseline"。这降低了后续 GAN T2I 研究的门槛——别人不用再从零搭一个 scale 起来的 GAN，可以直接在 Aurora 上做 ablation。这与 §4.5 的"刻板印象修正"结合，暗示后续可以系统研究"GAN 在 T2I 下的真实特性"（潜空间、可编辑性、3D 集成）。来源：§1.5（开源动机）+ §4.5（刻板印象修正）
  • 跨方法对比：sparse MoE 在 GAN 里 work（本文），在 diffusion 里也已经 work（如 Switch Diffusion）。但 Aurora 的 router 用了"条件 w"做 modulation——这个"让 router 看条件"的思路在 diffusion 里还没被充分探索，因为 diffusion 的条件注入主要靠 cross-attention 而非 router。可以尝试在 diffusion 的 MoE block 里也引入条件 modulated router。来源：§3.2 段落 4（条件 router）+ §2.3（sparse MoE 通用性）

一句话总结

Aurora 把 NLP 里的 sparse MoE 搬进 GAN 生成器，用"条件 modulated router"（router 同时看特征点和文本集成潜码 w）让 1.16B 参数的 GAN 在 64×64 上达到 6.45 zero-shot FID，性能比肩甚至超过同期 diffusion 方案且推理快；但有两个值得追问的点——(1) 论文与代码存在数字出入（z_dim、FID 6.2 vs 6.45），(2) 作者自己发现 T2I GAN 的潜空间不再平滑（Q9 未完全闭合），这反而动摇了 GAN 的一个传统卖点。整体值得跟，是一个诚实的、工程上可复现的 GAN T2I baseline，但若要扩展到 256+ 分辨率（Q10 未答）还需更多工作。

class TextHead(nn.Module):
    def __init__(self, ...):
        self.transformer = nn.Sequential(*[TextTransformerLayer(...) for _ in range(num_layers)])
        # num_layers_text_enc = 4（run_synthesize.py:113）

    def forward(self, text_feats, eot_ind):
        # text_feats: CLIP 输出 [bs, emb_length, dim]
        if self.align_ch:
            text_feats = self.align_dim(text_feats)
        bs, emb_length, dim = text_feats.shape
        device = text_feats.device

        text_feats = text_feats.permute(1, 0, 2)
        text_feats = self.transformer(text_feats)   # 叠 4 层可训练 transformer
        text_feats = text_feats.permute(1, 0, 2)

        # 用 eot_ind（CLIP 的 EOT 位置）切分全局 vs 局部
        length_ind = eot_ind[:, None, None].repeat(1, 1, dim).to(device)
        mask = torch.ones_like(text_feats, device=device)
        zero_fill = torch.zeros_like(text_feats, device=device)
        mask = torch.scatter(mask, dim=1, index=length_ind, src=zero_fill)
        local_feats = text_feats[mask.bool()].view(bs, emb_length-1, dim)  # t_seq
        global_feats = torch.gather(text_feats, dim=1, index=length_ind)   # t_g

        return global_feats[:, 0], local_feats

讲解： - CLIP 输出是一串 token feature，EOT（end-of-text）位置聚合了整句信息——这就是 global_feats (t_g)，其余 token 是 local_feats (t_seq) - self.transformer 是论文说的"在编码器顶部堆叠的可学习层"——num_layers_text_enc=4 层，对应论文 §3.1 "freeze CLIP + stack learnable layers" - ⚠️ 与论文的差异：论文没说叠几层，代码明示是 4 层 transformer（不是简单 MLP） - run_synthesize.py:96-98：clip_config = {'model_name':'ViT-L-14', 'pretrained':'openai', 'freeze_clip': True} 确认 CLIP ViT-L/14 + 冻结

class AttentionBlock(nn.Module):
    """self-attention + cross-attention + FFN(MoE)"""
    def forward(self, x, w, context=None):
        moe_loss = 0.0
        residual = x
        _, _, height, width = x.shape
        x = rearrange(x, 'b c h w -> b (h w) c')
        x = self.norm1(x)
        x = rearrange(x, 'b (h w) c -> b c h w', h=height, w=width)
        x, style = self.self_attn(x, w)                # 公式 (5) SelfAttn
        x = x + residual                                # residual

        x = rearrange(x, 'b c h w -> b (h w) c')
        if self.use_text_cond or self.use_w_cond:
            contexts = []
            if self.use_text_cond:
                assert context is not None
                contexts.append(context)               # t_seq
            if self.use_w_cond:
                w_ = self.align_dim(w)
                contexts.append(w_[:, None])
            contexts = torch.cat(contexts, dim=1)
            x = self.cross_attn(self.norm2(x), contexts) + x   # 公式 (6) CrossAttn

        if self.include_ff:
            out, moe_loss = self.ff(self.norm3(x), w)  # 公式 (7) FFN——但传了 w！
            x = out + x

        x = rearrange(x, 'b (h w) c -> b c h w', h=height, w=width)
        return x, style, moe_loss

讲解： - 依次执行 self_attn → cross_attn → ff，对应论文公式 (5)(6)(7) - ⚠️ 关键点：第 1918 行 self.ff(self.norm3(x), w) —— FFN 调用时把 w 也传进去了。这是 sparse MoE router 能用 w 做 modulation 的前提。论文 §3.2 段落 4 才点出 router 吃 w，但代码里 AttentionBlock 早就在调 FFN 时传 w——阅读代码能提前看出伏笔 - l2_attention=True（run_synthesize.py:129）对应论文"ℓ2-distance attention [29]" - attn_resolutions=['8','16','32','64']、attn_depth={"8":2,"16":2,"32":2,"64":1}（run_synthesize.py:121-122）——注意力块只在 ≥8 分辨率用，低分辨率纯卷积

class FeedForward(nn.Module):
    """Implements the feed-forward layer in attention layer."""
    def __init__(self, in_dim, mlp_ratio, dropout, w_dim, ...,
                 num_experts=-1):
        super().__init__()
        self.num_experts = num_experts

        if self.num_experts < 2:   # 普通单 FFN
            self.project_in = GEGLU(in_channels=in_dim,
                                    out_channels=in_dim * mlp_ratio, ...)
            self.proj_out = DenseLayer(in_channels=in_dim * mlp_ratio,
                                       out_channels=in_dim, ...)
            self.net = nn.Sequential(self.project_in,
                                     nn.Dropout(dropout),
                                     self.proj_out)
        else:                       # sparse MoE
            self.net = nn.ModuleList([])
            # 构建 N 个 expert，每个是 GEGLU + Dropout + Dense
            for _ in range(self.num_experts):
                project_in = GEGLU(in_channels=in_dim,
                                   out_channels=in_dim * mlp_ratio, ...)
                proj_out = DenseLayer(in_channels=in_dim * mlp_ratio,
                                      out_channels=in_dim, ...)
                net = nn.Sequential(project_in, nn.Dropout(dropout), proj_out)
                self.net.append(net)

            # router = 1x1 ModConv with w modulation
            self.routing = AdaptiveModulateConvLayer(
                                    in_channels=in_dim,
                                    out_channels=self.num_experts,
                                    modulate_dim=w_dim,        # 用 w 做 modulation
                                    kernel_size=1,
                                    num_kernels=1,
                                    demodulate=True,
                                    activation_type='lrelu',
                                    mtm=False)
            self.jitter_noise = 0.0
            self.expert_capacity = 262144   # 512*512

    def router(self, hidden_states, w):
        # float32 for stability
        original_dtype = hidden_states.dtype
        assert(original_dtype == torch.float32)

        # ... (jitter noise currently disabled)

        # reshape (B, HW, C) -> (B, C, H, W) for ModConv
        height = int(np.sqrt(hidden_states.shape[1]))
        width = int(np.sqrt(hidden_states.shape[1]))
        hidden_states = rearrange(hidden_states, 'b (h w) c -> b c h w',
                                  h=height, w=width)
        router_logits = self.routing(hidden_states, w)[0]   # 用 w 做 modulation
        router_logits = rearrange(router_logits, 'b c h w -> b (h w) c')

        router_probs = nn.functional.softmax(router_logits, dim=-1)
        expert_index = torch.argmax(router_probs, dim=-1)    # top-1 routing
        expert_index = torch.nn.functional.one_hot(expert_index,
                                                   num_classes=self.num_experts)

        # capacity mask: 超过 capacity 的 token 不路由
        token_priority = torch.cumsum(expert_index, dim=-2)
        expert_capacity_mask = token_priority <= self.expert_capacity
        expert_index = expert_index * expert_capacity_mask

        router_probs = torch.max(router_probs, dim=-1).values.unsqueeze(-1)
        return expert_index, router_probs, router_logits

    def cal_moe_loss(self, router_logits, expert_index):
        z_loss = router_z_loss_func(router_logits)              # 论文未提
        router_probs = nn.Softmax(dim=-1)(router_logits)
        aux_loss = load_balancing_loss_func(router_probs, expert_index)
        loss = 0.001 * z_loss + 0.001 * aux_loss
        return loss

    def forward(self, x, w=None):
        if self.num_experts < 2:
            return self.net(x), 0.0

        # Step 1: 路由
        router_mask, router_probs, router_logits = self.router(x, w)
        expert_index = torch.argmax(router_mask, dim=-1)

        # Step 2: 稀疏执行——每个 expert 只处理被路由到的 token
        next_states = x.clone()
        for idx, expert in enumerate(self.net):
            token_indices = router_mask[:, :, idx].bool()
            next_states[token_indices] = expert(x[token_indices])

        x = router_probs * next_states                          # gate weight
        moe_loss = self.cal_moe_loss(router_logits, expert_index)
        return x, moe_loss

def router_z_loss_func(router_logits):
    """Router z-loss from 'Designing Effective Sparse Expert Models'.
    Encourages router logits to remain small for stability.
    """
    num_groups, tokens_per_group, _ = router_logits.shape
    log_z = torch.logsumexp(router_logits, dim=-1)
    z_loss = log_z ** 2
    return torch.sum(z_loss) / (num_groups * tokens_per_group)


def load_balancing_loss_func(router_probs, expert_indices):
    """Switch Transformer load balancing loss.
    Implements equations (4)-(6) of Switch Transformer paper.
    Penalizes unbalanced routing across experts.
    """
    num_experts = router_probs.shape[-1]
    if expert_indices.dtype != torch.int64:
        expert_indices = expert_indices.to(torch.int64)
    if len(expert_indices.shape) == 2:
        expert_indices = expert_indices.unsqueeze(2)

    expert_mask = torch.nn.functional.one_hot(expert_indices, num_experts)
    expert_mask = torch.max(expert_mask, axis=-2).values
    expert_mask = expert_mask.to(torch.float32)

    # 每个专家被选中的 token 比例
    tokens_per_group_and_expert = torch.mean(expert_mask, axis=-2)
    # 每个专家的平均路由概率
    router_prob_per_group_and_expert = torch.mean(router_probs, axis=-2)

    # Switch Transformer 公式: f * P 的均值 × N^2
    return torch.mean(tokens_per_group_and_expert *
                      router_prob_per_group_and_expert) * (num_experts ** 2)

讲解： - load_balancing_loss_func 是标准 Switch Transformer loss：mean(tokens_per_expert * prob_per_expert) * N^2。对应论文 §3.3 说的 "MoE loss following Switch Transformer [15]" - router_z_loss_func 是 logsumexp(logits)^2 的均值——鼓励 logits 数值小，防 router 输出过大导致 softmax 饱和。论文未提及，是代码额外加的稳定性正则 - ⚠️ 与论文的差异：论文只提 load balancing loss，代码里 cal_moe_loss 实际是 0.001 * z_loss + 0.001 * aux_loss——两项相加，权重都极小。论文省略 z_loss 可能是因为它对最终 FID 影响小，但对训练稳定性有作用。复现时建议保留
