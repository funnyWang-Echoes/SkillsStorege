§1. Introduction

§1.1 段落 1（latent diffusion 的局限）

中文翻译： 扩散模型在高保真图像生成上取得了显著成功，提供了出色的质量和多样性。该领域的研究大致沿两个方向：latent diffusion 和 pixel diffusion。Latent diffusion 模型把生成拆成两阶段：先用 VAE 把图像压缩到紧凑的 latent 空间，再由扩散模型在该空间内运行。然而，它们的性能很大程度上受限于 VAE 的重建质量和 latent 分布。训练 VAE 通常需要对抗监督或额外监督，这可能不稳定，并会造成不可避免的低层伪影。

§1.1 段落 2-3（合并：pixel diffusion 的挑战 + Fig.2）

中文翻译：

（段落 2）Pixel diffusion 通过直接对原始像素端到端建模，避开了上述依赖 VAE 的局限。这能实现更优的分布学习，并消除不完美 VAE 压缩带来的伪影。然而，在庞大的像素空间中 jointly 建模复杂高频信号和低频语义是困难的。如 Fig. 2 (a) 所示，传统方法通常依赖单个模型（如 DiT）在每个 timestep 从单尺度输入学习这两个分量。复杂的高频信号，尤其是高频噪声，很难学习；它们还会分散 DiT 学习低频语义的注意力。如 Fig. 2 (c) 所示，这种范式导致 DiT 输出含噪、图像质量下降。

（段落 3）此前的工作表明：从高分辨率输入重建高频信号、从低分辨率输入建模低频语义，更为有效。另一些研究显示，transformer 倾向于擅长捕捉低频语义但在高频信号上表现吃力。因此我们提出解耦高低频分量的生成。如 Fig. 2 (b) 所示，DeCo 让 DiT 专攻下采样输入的低频语义建模；语义线索随后被融入轻量 pixel decoder 来重建高频信号。换言之，pixel decoder 以 DiT 的低频语义为条件，用高分辨率输入预测像素速度。这一新范式解放了 DiT 专攻语义建模，并允许更专门化的细节生成。

图 2：(a) 传统基线用单个 DiT 同时建模低频语义和高频信号；(b) DeCo 框架，轻量 pixel decoder 专注高频重建，DiT 建模低频语义；(c) 解耦后 DiT 输出的低频语义特征更好、图像质量更高。

§1.1 段落 4（frequency-aware FM loss）

中文翻译： 为进一步强调视觉显著的频率、抑制感知上不显著的高频分量，我们引入受 JPEG 启发的 frequency-aware Flow-Matching (FM) loss。标准 FM loss 对所有频率一视同仁，而我们的 frequency-aware 变体先用离散余弦变换 (DCT) 把像素速度变到频域，再为每个频带分配自适应权重。这些自适应权重来自 JPEG 量化表，后者编码了不同频率视觉重要性的稳健先验。通过强调视觉显著频率、抑制高频噪声，该损失简化了优化景观并增强视觉保真度。值得注意的是，我们的动机与并发工作 JiT 高度一致：JiT 给出显式公式解耦高频噪声与干净图像，而我们的 DeCo 通过 pixel decoder 提供高频捷径，实现隐式频率解耦。更多讨论见附录 A。

§2. Related Work

§2.1 段落 1（Latent Diffusion）

中文翻译： Latent diffusion 在 VAE 学到的紧凑 latent 空间中训练扩散模型。相比原始像素空间，latent 空间显著降低了空间维度，降低了学习难度和计算开销。因此 VAE 已成为现代扩散模型的基础组件。然而，训练 VAE 常涉及对抗目标和感知监督，使整体流程复杂化；训练不善的 VAE 会产生解码伪影，限制 latent diffusion 的泛化。早期 latent diffusion 主要用 U-Net 架构。开创性的 DiT 把 transformer 引入扩散模型，替换了 U-Net；SiT 进一步用线性流扩散验证了 DiT。后续工作通过表示对齐和联合优化增强 latent diffusion：REPA 把中间特征与预训练 DINOv2 对齐以学习更好的低频语义，这与我们的框架兼容，并同时在我们的 baseline 和最终 DeCo 中使用；REPA-E 试图端到端联合优化 VAE 和 DiT，但该方法可能因扩散损失而训练崩溃——持续变化的 latent 空间导致去噪目标不稳定。相比之下，pixel diffusion 在固定空间去噪，确保目标一致、训练稳定。

§2.1 段落 2（Pixel Diffusion）

中文翻译： 由于像素空间的巨大维度，pixel diffusion 的进展比其 latent 对应物慢得多。早期像素扩散模型通常依赖长残差连接，可能妨碍可扩展性。近期尝试把扩散过程在不同分辨率尺度上分块以降低计算开销：早期方法把扩散过程拆成多个分辨率阶段——Relay Diffusion 为每个尺度训练单独模型，导致更高开销和两阶段优化；PixelFlow 用一个模型跨所有尺度，但需要复杂的去噪调度，拖慢推理。替代路线探索不同架构：FractalGen 通过递归应用原子模块构建分形生成模型；TarFlow 引入基于 transformer 的归一化流直接建模和生成像素；近期 PixNerd 用 DiT 为每个 patch 预测神经场参数，类似测试时训练地渲染像素速度；JiT 预测干净图像把生成锚定到低维数据流形；PixelGen 引入感知监督简化像素扩散训练。

§2.1 段落 3（Frequency-Decoupled Image Generation）

中文翻译： 多尺度级联方法可大致视为一种时序频率解耦——早期步生成低频语义、后期步细化高频细节。然而这些方法仍用单一模型/架构在每个 timestep 学习所有频率，高频信号依然存在，高频噪声可能干扰低频语义学习；它们还依赖复杂去噪调度和小 patch size，降低训练或采样效率。我们的 DeCo 在架构中引入显式频率解耦：不是跨 timestep 分离不同频率，而是在每个 timestep 内端到端地同时处理它们。近期的两阶段工作 DDT 在压缩 latent 空间探索单尺度频率解耦，表明频率解耦即便在压缩空间仍重要。与 DDT 不同，我们的 DeCo 是面向像素扩散的多尺度设计；我们的 decoder 使用 attention-free 的线性层而非 DDT 的 attention-based DiT block，在大尺度输入上更高效。

§3. Method

§3.1 段落 1（CFM 预备 + baseline）

中文翻译： 本部分先回顾 baseline pixel diffusion 中的条件流匹配（conditional flow matching），再介绍我们的频率解耦像素扩散框架。

条件流匹配。 条件流匹配提供了一个连续时间生成建模框架，它学习一个速度场 \( v_{\theta}(x,t,y) \)，把样本从简单先验分布（如高斯）传输到以标签 \( y \) 和时间 \( t \) 为条件的数据分布。给定由干净图像 \( x_0 \) 和噪声 \( x_1 \) 之间插值定义的前向轨迹 \( x_t \)，条件流匹配的目标是把模型预测的速度 \( v_{\theta}(x_t,t,y) \) 匹配到 ground-truth 速度 \( v_t \)：

$$ \mathcal{L}_{\mathrm{FM}} = \mathbb{E}_{x_t,t,y}\left[\left\|v_{\theta}(x_t,t,y) - v_t\right\|^2\right] \tag{1} $$

其中轨迹 \( x_t \) 的线性插值定义为 \( x_t = (1-t)x_0 + t x_1 \)（式 2），ground-truth 速度 \( v_t \) 可由 \( x_t \) 的时间导数推出，即 \( v_t = x_t' = x_1 - x_0 \)（式 3）。

在 pixel diffusion baseline 中，轨迹 \( x_t \) 通常先被 patch embedding 层 patchify 成 token（而非用 VAE 下采样）。在我们的 baseline 和 DeCo 实验中，DiT 输入的 patch size 均为 16。在 baseline 中，patchify 后的轨迹 \( \bar{x}_t \) 被送入 DiT，由 unpatchify 层预测像素速度。DiT 需要同时建模高频信号和低频语义；高频信号（尤其高频噪声）难以建模，会分散 DiT 学习低频语义的注意力。

§3.1 段落 2（DeCo overview + Fig.3）

中文翻译： DeCo。 为把高频生成与低频语义建模分离，我们提出频率解耦框架 DeCo。如 Fig. 3 所示，DiT 被用于从下采样的低分辨率输入 \( \bar{x}_t \) 生成低频语义 \( c \)：
$$ x_{\mathrm{low}} = \mathrm{DiT}(\bar{x}_t, t, y) \tag{4} $$

其中 \( t \) 为时间、\( y \) 为标签或文本 prompt。如 §3.2 所述，轻量 pixel decoder 随后以 DiT 的低频语义 \( c \) 为条件，用全分辨率稠密输入 \( x_t \) 生成额外的高频细节，预测最终像素速度：

$$ v_{\theta}(x_t, t, y) = \mathrm{Dec}(x_t, t, x_{\mathrm{low}}) \tag{5} $$

这一新范式利用 pixel decoder 生成高频细节，解放 DiT 专攻语义建模。该解耦把不同频率的建模拆分到不同模块，带来更快训练和更好视觉保真度。

为进一步强调视觉显著频率、忽略不显著的高频分量，我们引入 frequency-aware FM Loss \( \mathcal{L}_{\mathrm{FreqFM}} \)（见 §3.3）。该损失用来自 JPEG 感知先验的自适应权重重新加权不同频率分量。结合标准像素级 flow-matching loss 和 baseline 的 REPA 对齐损失，最终目标为：

$$ \mathcal{L} = \mathcal{L}_{\mathrm{FM}} + \mathcal{L}_{\mathrm{FreqFM}} + \mathcal{L}_{\mathrm{REPA}} \tag{6} $$

图 3：DeCo 框架总览。DiT 在下采样输入上运行建模低频语义，轻量 pixel decoder 在 DiT 语义引导下生成高频细节。

§3.2 段落 1（Pixel Decoder 概览 + Dense Query）

中文翻译： 如 Fig. 3 所示，pixel decoder 是一个轻量的 attention-free 网络，由 N 个线性 decoder block 和若干线性投影层组成。所有操作都是局部且线性的，能在没有自注意力开销的情况下高效建模高频。

稠密查询构造（Dense Query Construction）。 pixel decoder 直接以全分辨率噪声图像为输入，不做下采样。所有噪声像素与对应的位置编码 pos 拼接，经 \( W_{\mathrm{in}} \) 线性投影成稠密查询向量 \( h_0 \)：

$$ h_0 = W_{\mathrm{in}}(\mathrm{Concat}(x_t, pos)) \tag{7} $$

其中 \( h_0 \in \mathbb{R}^{B\times H\times W\times d} \)，H、W 为原始图像高宽（如 256），d 为 pixel decoder 的隐藏维度（如 32）。相关消融见 Tab. 4 (c)(d)。

§3.2 段落 2（Decoder Block + AdaLN 调制）

中文翻译： Decoder Block。 对每个 decoder block，DiT 输出 \( x_{\mathrm{low}} \) 被线性上采样并 reshape 以匹配 \( x_t \) 的空间分辨率，得到 \( x_{\mathrm{low}}^{\mathrm{up}} \)。一个 MLP 随后生成 AdaLN 的调制参数 \( \alpha, \beta, \gamma \)：
$$ \alpha, \beta, \gamma = \mathrm{MLP}(\sigma(x_{\mathrm{low}}^{\mathrm{up}} + t)) \tag{8} $$

其中 \( \sigma \) 为 SiLU 激活。我们使用 AdaLN-Zero 在每个 block 内调制稠密 decoder query：

$$ h_N = h_{N-1} + \alpha * (\mathrm{MLP}((1+\gamma)*h_{N-1} + \beta)) \tag{9} $$

其中 MLP 含两个带 SiLU 的线性层。

速度预测（Velocity Prediction）。 最后，一个线性投影加 rearrangement 操作把 decoder 输出映射到像素空间，得到预测速度 \( v_\theta(x_t, t, y) \)。该速度包含 pixel decoder 生成的高频细节和来自 DiT 的语义线索。

§3.3 段落 2（Perceptual Weighting + 损失定义）

中文翻译： 感知加权（Perceptual Weighting）。 为强调视觉显著频率、抑制不显著频率，我们用 JPEG 量化表作为视觉先验生成自适应权重。量化间隔更小的频率在感知上更重要。因此我们用质量 \( q \) 下缩放量化表 \( Q_{\mathrm{cur}} \) 的归一化倒数作为自适应权重 \( w \)，即 \( w = \frac{1}{Q_{\mathrm{cur}}} / \mathbb{E}(\frac{1}{Q_{\mathrm{cur}}}) \)。当质量 \( q \) 在 50 到 100 之间时，缩放量化表 \( Q_{\mathrm{cur}} \) 可按 JPEG 预定义规则获得：
$$ Q_{\mathrm{cur}} = \max\left(1, \left\lfloor\frac{Q_{\mathrm{base}}\cdot(100-q)+25}{50}\right\rfloor\right) \tag{11} $$

其中 \( Q_{\mathrm{base}} \) 为 JPEG 规范定义的标准基础量化表。有了自适应权重 \( w \)，frequency-aware FM loss 定义为：

$$ \mathcal{L}_{\mathrm{FreqFM}} = \mathbb{E}_{x_t,t,y}\left[w\left\|\mathbb{V}_\theta - \mathbb{V}_t\right\|^2\right] \tag{12} $$

§A. Comparison with JiT（理论附录，完整精读）

§A.1 段落 1（隐式 vs 显式解耦 + Fig.7）

中文翻译： DeCo 的关键思想是通过 pixel decoder 提供高频捷径，实现隐式频率解耦。高频分量包括噪声输入 \( x_t \) 和高频细节 \( \Delta \)。在标准 DiT 中，高频信号在 patchification 和深层 transformer 层后倾向于被抑制。由于 pixel decoder 直接建模原始输入的每个像素，DeCo 能自动利用这一高频捷径、把高频建模卸载到 pixel decoder，使 DiT 专注低频分量 \( x_{\mathrm{low}} \)。与 JiT 显式预测 \( x_0 = x_{\mathrm{low}} + \Delta \) 不同，我们的 DiT 隐式建模 \( x_{\mathrm{low}} \)（排除难的高频细节 \( \Delta \)）。pixel decoder 在 \( x_{\mathrm{low}} \) 引导下，只需拟合一个简单的 JiT 式公式 \( v_\theta = \frac{x_{\mathrm{low}} + (\Delta - x_t)}{t} \)。这种分离是端到端学得的。Fig. 2(c) 的可视化、Fig. 4 的频谱分析、以及 Tab. 1 中优于 "JiT+REPA" 的性能验证了我们的隐式解耦。

§D. Quantization Tables（理论附录，完整精读）

§D.1 段落 1（量化表原理）

中文翻译： 在我们的 DeCo 中，用缩放 JPEG 量化表的归一化倒数作自适应权重来强调不同频率分量。这些表是 JPEG 压缩标准的核心组件，基于人类视觉系统 (HVS) 特性设计。

如 §3.3 所述，量化表是 8×8 矩阵，决定 DCT 后每个频率系数的压缩级别。JPEG 标准用两张独立表：一张用于亮度 (Y)、一张用于色度 (Cb/Cr)。该设计基于人类感知的关键特性：人眼对低频分量比对高频分量敏感得多；人眼对亮度变化比对色度变化敏感。基于大量实验，标准基础量化表 \( Q_{\mathrm{base}} \) 对低频系数用更小值（更细量化间隔，感知更重要），对高频用更大值（更粗间隔，信息损失不易察觉）；亮度表一般比色度表值更小。这些基础表可用质量因子 \( q \) 缩放。

由于更小的量化步意味着该频率对人眼更显著，我们用缩放量化表的归一化倒数作自适应权重，即 \( \frac{1}{Q_{\mathrm{cur}}} \) 并归一化。这让我们在 frequency-aware FM loss 中给视觉更重要的频率分量更高权重。

即时分析： - 这段在说什么：解释 JPEG 量化表的 HVS 依据——低频更重要、亮度比色度敏感 - 【回应 Q4】：JPEG 量化表的设计基于通用 HVS，对自然图像普适；但对极端分布（医学、纹理）可能不优——论文未讨论此局限。Q4 标记为开放 ✗ - 有没有问题：理论解释清晰。但没有讨论"若数据分布与自然图像差异大，JPEG 先验是否仍适用"这一局限

举一反三

  • 机制迁移：频率解耦到视频扩散（来源：§3.1-§3.2 + §3.4）。DeCo 的"DiT 建模低频、轻量 decoder 建模高频"可迁移到视频扩散：DiT 在下采样时空 token 上建模运动语义（低时空频），一个轻量时空 decoder 在全分辨率上补高频外观细节。为什么能迁移：视频的高频主要是纹理/边缘，低频是运动/结构，与图像的频率分布同构。最小验证实验：在 UCF-101 或 SkyTimelapse 上，把现有视频 DiT（如 MagCache）的最后两个 block 换成 3 层 attention-free 时空 decoder，d=32，patch=1，训 50k 步，对比 FVD 和训练 s/it。看 FVD 是否降、s/it 是否持平。

  • 方法改进：JPEG 先验换成任务特定频域加权（来源：§3.3 + Q4 局限）。DeCo 的 FreqFM loss 用 JPEG 量化表加权，但对非自然图像（医学、遥感）可能不优。可把 \( w = 1/Q_{cur} \) 换成数据驱动学习：用一个小网络从训练集统计出频带重要性。最小验证实验：在 ImageNet 上先跑 DeCo 200k 步得 FID 31.35 基线；然后把 freq_w 换成可学习参数（8×8×3 = 192 个标量），同样 200k 步，看 FID 是否 < 31.35。若可学习权重收敛后与 JPEG 表差异大，说明 JPEG 先验非最优。

  • 方法改进：decoder 深度自适应（来源：§4.4 Tab.4(b) + Q9）。论文固定 N=3 层 decoder，但不同分辨率/不同 timestep 的"高频复杂度"不同——早期去噪步高频噪声多、后期步高频细节多。可让 decoder 深度随 timestep 自适应（早步多层、晚步少层）或随分辨率缩放。最小验证实验：在 ImageNet 256×256 上，把 decoder 深度设为 N = 2 + round(2*(1-t))（t∈[0,1]，早步 N=4、晚步 N=2），训 200k 步对比 FID 与固定 N=3 的 31.35。看自适应是否带来增益。

  • 机制迁移：频率解耦到 3D 生成（来源：§3.4 DCT 频谱分析 + Fig.4）。DeCo 的 DCT 频谱诊断方法（Fig.4）可迁移到 3D 扩散（如点云/NeRF 扩散）：用 3D DCT 分析 DiT 输出的频谱，若高频能量过高则加频率解耦。最小验证实验：在 ShapeNet 上跑一个点云扩散基线，用 3D DCT 测 DiT 输出频谱；若高频能量占比 > 40%，引入 3D pixel decoder（attention-free）试解耦，看 Chamfer Distance 是否改善。

一句话总结

DeCo 用"DiT 建低频语义 + 轻量 attention-free decoder 补高频细节 + JPEG 先验加权频域 loss"三件套，在像素扩散上做到 ImageNet 1.62 FID、GenEval 0.86，训练效率比 baseline 快 10×，是与 latent diffusion 同台竞技的端到端像素扩散新基线——思路清晰、消融扎实、代码全开源，值得跟。唯一需留意的是收益来自"架构+REPA+FreqFM"三者协同，单独架构解耦反而退步。

代码块 1: Pixel Decoder 稠密查询构造（NerfEmbedder）

对应论文： §3.2 Dense Query Construction（式 7） 文件： src/models/transformer/dit_c2i_DeCo.py:212-248

class NerfEmbedder(nn.Module):
    def __init__(self, in_channels, hidden_size_input, max_freqs):
        super().__init__()
        self.max_freqs = max_freqs
        self.hidden_size_input = hidden_size_input
        self.embedder = nn.Sequential(
            nn.Linear(in_channels+max_freqs**2, hidden_size_input, bias=True),
        )

    @lru_cache
    def fetch_pos(self, patch_size, device, dtype):
        pos_x = torch.linspace(0, 1, patch_size, device=device, dtype=dtype)
        pos_y = torch.linspace(0, 1, patch_size, device=device, dtype=dtype)
        pos_y, pos_x = torch.meshgrid(pos_y, pos_x, indexing="ij")
        pos_x = pos_x.reshape(-1, 1, 1)
        pos_y = pos_y.reshape(-1, 1, 1)
        # DCT 基函数作位置编码（天然支持任意分辨率）
        freqs = torch.linspace(0, self.max_freqs, self.max_freqs, dtype=dtype, device=device)
        coeffs = (1 + freqs_x * freqs_y) ** -1
        dct_x = torch.cos(pos_x * freqs_x * torch.pi)
        dct_y = torch.cos(pos_y * freqs_y * torch.pi)
        dct = (dct_x * dct_y * coeffs).view(1, -1, self.max_freqs ** 2)
        return dct

    def forward(self, inputs):
        B, P2, C = inputs.shape
        patch_size = int(P2 ** 0.5)
        dct = self.fetch_pos(patch_size, inputs.device, inputs.dtype)
        dct = dct.repeat(B, 1, 1)
        inputs = torch.cat([inputs, dct], dim=-1)  # 像素 + DCT pos
        inputs = self.embedder(inputs)             # W_in 线性投影
        return inputs

讲解： - NerfEmbedder 实现论文式 (7) 的 \( h_0 = W_{\mathrm{in}}(\mathrm{Concat}(x_t, pos)) \) - 位置编码 pos 用 DCT 基函数（torch.cos(...*π)）而非通常的 sin/cos——这与论文的频率主题呼应，且 lru_cache + 按分辨率生成天然支持任意分辨率（回答了 Q12：512×512 微调时位置编码自动适配） - max_freqs=8 → 64 维 pos，与像素通道拼接后线性投影到 hidden_size_input=32（对应论文 d=32） - 命名带 "Nerf" 是历史遗留（PixNerd 谱系），实际就是 pixel decoder 的输入嵌入层

class ResBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.channels = channels
        self.in_ln = nn.LayerNorm(channels, eps=1e-6)   # 论文未提的额外 norm
        self.mlp = nn.Sequential(
            nn.Linear(channels, channels, bias=True),
            nn.SiLU(),
            nn.Linear(channels, channels, bias=True),
        )
        self.adaLN_modulation = nn.Sequential(
            nn.SiLU(),                                    # 对应式(8)的 σ=SiLU
            nn.Linear(channels, 3 * channels, bias=True)  # 输出 α,β,γ
        )

    def forward(self, x, y):
        # y = 条件（DiT 输出经上采样 + t）
        shift_mlp, scale_mlp, gate_mlp = self.adaLN_modulation(y).chunk(3, dim=-1)
        h = modulate(self.in_ln(x), shift_mlp, scale_mlp)  # (1+γ)*norm(x)+β
        h = self.mlp(h)
        return x + gate_mlp * h                            # 残差 + α 门控

其中 modulate(x, shift, scale) = x * (1 + scale) + shift（文件第 11 行）。

讲解： - ResBlock 实现论文式 (8)(9) 的 AdaLN-Zero 调制：y（条件）→ adaLN_modulation（含 SiLU，即式 8 的 σ）→ 输出 (shift, scale, gate) = (β, γ, α) - forward 里的 modulate(self.in_ln(x), ...) 对应式 (9) 的 (1+γ)*h_{N-1}+β，但代码多了一层 LayerNorm(in_ln)——论文式 (9) 未写这个 norm，是实现与论文的细微差异 - return x + gate_mlp * h 对应式 (9) 的 h_{N-1} + α*(...)，gate_mlp 就是 α - SimpleMLPAdaLN（同文件 334-415 行）把 N=3 个 ResBlock 堆起来，外加 cond_embed（把 DiT 的 hidden_size 投影到 patch_size²×decoder_dim）和 FinalLayer——这就是论文的"Pixel Decoder"

代码块 3: DeCo 前向（DiT → Decoder 数据流）

对应论文： §3.1 Overview（式 4-5） 文件： src/models/transformer/dit_c2i_DeCo.py:488-510（PixNerDiT.forward）

def forward(self, x, t, y, s=None, mask=None):
    B, _, H, W = x.shape
    pos = self.fetch_pos(H//self.patch_size, W//self.patch_size, x.device)
    # 1) patchify 成 token（patch_size=16，下采样）
    x = torch.nn.functional.unfold(x, kernel_size=self.patch_size,
                                    stride=self.patch_size).transpose(1, 2)
    t = self.t_embedder(t.view(-1)).view(B, -1, self.hidden_size)
    y = self.y_embedder(y).view(B, 1, self.hidden_size)
    c = nn.functional.silu(t + y)
    # 2) DiT 建模低频语义（num_cond_blocks=22 层）
    if s is None:
        s = self.s_embedder(x)
        for i in range(self.num_cond_blocks):
            s = self.blocks[i](s, c, pos, mask)      # 式(4): x_low = DiT(x̄_t)
        s = nn.functional.silu(t + s)
    # 3) Pixel decoder 预测速度（式5）
    batch_size, length, _ = s.shape
    x = x.reshape(batch_size*length, self.in_channels, self.patch_size**2)
    x = x.transpose(1, 2)
    s = s.view(batch_size*length, self.hidden_size)
    x = self.x_embedder(x)        # NerfEmbedder：稠密 query（式7）
    x = self.dec_net(x, s)        # SimpleMLPAdaLN：3 层 ResBlock（式8-9）
    # 4) unpatchify 回像素空间
    x = x.transpose(1, 2).reshape(batch_size, length, -1)
    x = torch.nn.functional.fold(x.transpose(1, 2).contiguous(), (H, W),
                                  kernel_size=self.patch_size, stride=self.patch_size)
    return x

讲解： - 数据流完全对应论文式 (4)(5)：x_t → patchify（patch=16）→ DiT（num_cond_blocks=22 层）得 s=x_low → dec_net（pixel decoder）以 s 为条件预测速度 - 关键细节：decoder 的输入 x 是 patchify 后的（patch_size=16），但 NerfEmbedder 内部对每个 patch 的 patch_size**2=256 个像素分别建 query——所以"全分辨率 dense query"是 patch 内的全分辨率，不是整图全分辨率。这与论文表述"直接以全分辨率噪声图像为输入"略有出入：实际是 patch 级全分辨率。Tab.4(c) 的 patch_size 消融指的是 decoder 的 patch_size（=1），不是 DiT 的 patch_size（=16） - num_cond_blocks=22（DiT 层）+ num_blocks-num_cond_blocks=3（decoder 层）= 25 总深度（config 确认）。这澄清了 Q7：论文说"替换最后两个 DiT block"实际是替换成 3 层 decoder

def _create_dct_matrix(self, N: int):
    import math
    n = torch.arange(N, dtype=torch.float32)
    k = torch.arange(N, dtype=torch.float32).unsqueeze(1)
    C = torch.cos(math.pi * (2*n + 1) * k / (2.0 * N))
    alpha = torch.sqrt(torch.tensor(2.0) / N) * torch.ones(N)
    alpha[0] = math.sqrt(1.0 / N)
    C = alpha.unsqueeze(1) * C
    return C  # (N,N) 标准 DCT-II 基矩阵

def _rgb2ycbcr(self, x: torch.Tensor):
    r, g, b = x[:,0:1], x[:,1:2], x[:,2:3]
    y  = 0.299*r + 0.587*g + 0.114*b
    cb = -0.168736*r - 0.331264*g + 0.5*b
    cr = 0.5*r - 0.418688*g - 0.081312*b
    return torch.cat([y, cb, cr], dim=1)

@torch.compile()
def _dct(self, x: torch.Tensor):
    bs = self.block_size  # 8
    B, C, H, W = x.shape
    pad_h, pad_w = (-H) % bs, (-W) % bs
    if pad_h > 0 or pad_w > 0:
        x = F.pad(x, (0,pad_w,0,pad_h), mode="reflect")
    blocks = x.unfold(2, bs, bs).unfold(3, bs, bs)  # 分 8×8 块
    blocks = blocks.contiguous().view(-1, bs, bs)
    Cmat = self.dct_mat.to(x.device, x.dtype)
    dct_flat = torch.matmul(Cmat.unsqueeze(0), blocks)
    dct_flat = torch.matmul(dct_flat, Cmat.t().unsqueeze(0))  # 二维 DCT
    return dct_flat.view(B, C, Bh, Bw, bs, bs)

讲解： - _create_dct_matrix 构造标准 8×8 DCT-II 基矩阵，alpha 处理 DC 分量的归一化——完全遵循 JPEG - _rgb2ycbcr 用 ITU-R BT.601 标准转换，Y/Cb/Cr 分开以便用不同量化表（Y 用亮度表、Cb/Cr 用色度表） - _dct 用 unfold 分块 + 矩阵乘法实现 2D DCT，比循环快；@torch.compile() 加速。分块大小 block_size=8 与 JPEG 一致 - 这三段实现论文式 (10) 的 \( \mathcal{T} \)，输入 (B,C,H,W) 速度，输出 (B,C,Bh,Bw,8,8) 频域系数

def _build_freq_weight(self, quality=85, mode='inv_gamma', gamma=1.0):
    # JPEG 标准亮度/色度基础量化表
    lum_q = torch.tensor([
        [16,11,10,16,24,40,51,61],
        [12,12,14,19,26,58,60,55],
        # ... 8x8
    ], dtype=torch.float32)
    chr_q = torch.tensor([
        [17,18,24,47,99,99,99,99],
        # ... 8x8
    ], dtype=torch.float32)

    def scale_q(base_q, quality):
        q = max(1, min(100, int(quality)))
        if q < 50:
            scale = 5000 / q
        else:
            scale = 200 - 2*q   # 对应式(11)的 (100-q)*2
        return torch.floor((base_q*scale + 50)/100).clamp(1,255)

    Q_y = scale_q(lum_q, quality)
    Q_cbcr = scale_q(chr_q, quality)

    def q_to_weight(Q):
        if mode == 'inv':
            w = 1.0 / Q                          # 论文式: 1/Q_cur
        elif mode == 'inv_gamma':
            w = (Q.mean()/Q) ** gamma            # gamma=1 时等价于 1/Q 归一化
        return w / w.mean()                      # 归一化：除以均值
    w = torch.stack([q_to_weight(Q_y),
                     q_to_weight(Q_cbcr),
                     q_to_weight(Q_cbcr)], dim=0)
    return w.unsqueeze(0).unsqueeze(2).unsqueeze(3)  # (1,3,1,1,8,8)
