# 科研写作质量原则（完整版）

本文件提炼自 `E:\WSL and SSH remote download\geo_project` 中多篇幅的高质量科研进展写作范例（用户确认质量高，可借鉴其写法）。这些原则用于约束 `research-progress` skill 生成科研进展类文档时的**质量下限**。

---

## 一、数据驱动，量化优先

一切结论尽量用数字和表格说话。

- 劣：「模型在前期表现较好，后期有所下降。」
- 优：「DirectMLP 在 1850–1900 达到 RMSE 0.685℃，较 per-site OLS 基线 0.707℃ 降低 3.15%；到 1961–2000 升至 0.801℃，反超基线 +2.92%。」

表格优于长段落。关键对比用「方法 | 指标 | vs 基线 | 结论」四列表。

---

## 二、诚实报告 negative result

失败的实验和被排除的假设，往往比报喜更有科研价值。要写「排除了什么、为什么不行」。

- 显式列出「排除了什么」表：假设 / 检验方式 / 结论。
- 失败的实验要给出根因分析，而不是简单说「未能超越基线」。
- 例：「M2_north 的结论不是 DL 的失败，而是问题结构决定：479 个低 SNR 稀疏代理包含的线性可提取信息量，本身就不足以为 6912 个格点提供独立约束。direct_ridge 在此信息条件下就是可达上限。」

---

## 三、公平基线 + 统计显著性

- 对比必须是**同口径**：同样的数据集、同样的校准/训练窗口、同样的目标真值、同样的评估时段。
- 明确标注 baseline 是否「公平」。不公平 baseline 得出的「胜出」要单独说明为什么不公平。
- 优先用**置信区间 / 多次 seed 均值**，而非单点数字。例：「Cross-Attn field_area_CE = 0.138，95% CI [0.076, 0.211]，下界 > 0 但显著低于 direct_ridge 0.271。」
- 警惕「劣质基线假象」：某个方法看起来赢，可能只是因为对比的 baseline 太弱。引入公平基线后结论可能反转——这正是高质量工作的体现。

---

## 四、机制分析重于堆叠结果

不只报数字，要解释 *为什么*。

- 给出「为什么只在部分场景有效」的深层机制：是数据量不足？信息量瓶颈？归纳偏置与问题结构不匹配？训练/测试气候态偏移？
- 用「被推翻的假象」表做自我审查：表面胜出 → 被什么进一步实验推翻 → 真相。
- 用结构图（代码块里的树状文本）说明失败的根本结构。

---

## 五、结构化、可追溯

推荐通用骨架（探索性实验小结 / 综合报告适用）：

```
标题（含核心结论一句话）
元信息（日期 / 数据口径 / 脚本路径 / 证据等级）
1. 背景
2. 动机（为什么要做这个实验 / 解决什么风险）
3. 实验（数据 / 方法 / 基线 / 评估段）
4. 结果（核心表 + 解读）
5. 机制分析（为什么）
6. 结论（每条结论对应证据）
7. 讨论（相关工作 / 局限 / 未来方向）
8. 参考文献
```

月报类用 `templates/科研月报-导师.md` 的结构；文献类用 `templates/文献分析.md` 的结构。

---

## 六、去 AI 味硬规则

- 温度 0.3–0.5，少煽情修饰词。
- 单项汇报 ≤ 150 字。
- 强制具象参数：温度、精度、耗时、RMSE、样本量、seed 数等。
- 禁用高频过渡词：「值得注意的是」「正如我们所知」「总而言之」「综上所述」「此外」「值得一提的是」。用分段 + 图标（✅ ❌ ⚠️ ★）代替。
- 未发表公式 / 核心代码 / 专利，用「算法 A」「化合物 B」代替。

---

## 七、证据等级标注（可选但推荐）

对关键结论标注证据等级，提升可信度：

- **A**：多配置 × 多 seed × bootstrap 置信区间，统计显著。
- **clean ✅**：已修复数据泄漏 / 时间隔离等风险，结论可靠。
- **⚠️**：CI 跨零或不显著，需谨慎。
- **❌**：显著差于基线 / 气候态。

例：「★★ 与旧 README 记录的 0.6963（-9.70%）不同：旧数字是 fold-0 单次结果（源于 bug），本次为修复后的 5 折均值。」——这种「主动纠正旧错误」的态度是高质量标志。
